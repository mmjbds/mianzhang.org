GitHub Paper Policy

Index everything public. Mirror PDFs only when justified.

The public GitHub layer stores small, text-first metadata: DOI, record URL, version, publication date, manifest, checksums, evidence level, and forbidden claims. Large binaries, non-public review files, cloud returns, and private release packages stay outside the public git tree.

Canonical

Zenodo first

Cite DOI landing pages and version records. Use GitHub pages as navigation, not authority replacement.

Allowed

Text index

Keep JSONL, CSV, README cards, claim boundaries, hash receipts, and release notes in GitHub.

Optional

Release assets

Mirror a few public, license-clear PDFs in GitHub Releases only when it helps readers.

Boundary

Private/sensitive

Sensitive operational material is outside the public research archive.

Boundary

Public DOI records establish priority and inspectability. They are the authority for public citation and version tracking.

Venue-specific materials should use the prepared route for that venue; the public paper index remains focused on citable public records.
