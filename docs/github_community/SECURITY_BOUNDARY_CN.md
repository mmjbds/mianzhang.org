# 安全边界

安全报告遵循“防御性、授权范围内、最小公开信息”原则。

## 公开 issue 可以包含

- 公开 README、公开 demo 或公开页面的安全边界表述问题。
- claim 被写得过宽，可能被误读为生产安全保证。
- 证据污染、credit leak、权限边界或 no-go 说明不足。
- 不含敏感细节的配置建议、文档建议或公开链接修正。

## 公开 issue 不得包含

- API key、token、密码、cookie、私有配置或真实账户。
- 客户数据、真实交易日志、私有日志、内部 prompt 或私有 agent 编排。
- 可直接复用的 exploit、绕过步骤、攻击脚本或第三方攻击目标。
- 会议匿名材料、审稿私有页面、邮件截图或可去匿名化信息。

## 私有报告应包含

- 受影响的公开文件、页面、commit、demo 或 registry。
- 风险类型、影响范围和最小触发条件。
- 使用公开样例或合成样例的复现描述。
- 建议修复方式，例如删除链接、收窄 claim、添加 no-go 或更新模板。

如果材料无法在不暴露敏感信息的情况下说明，请先走私有报告，不要为了完整性把敏感材料贴到公开仓库。
