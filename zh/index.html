核心定位

中文区不再另起炉灶，先做同域名下的可信入口。

现在最科学的做法不是单独建一个中文网站，也不是把 Hugging Face 扩成中文站。中文内容先放在 mianzhang.org/zh：同一个域名、同一套 sitemap、同一套证据边界，成本低，维护轻。

微信、知乎、B 站、微博只做分发，不做权威归档。需要引用论文时回到 Zenodo，需要挑战主张时回到 GitHub issue，需要看技术镜像时回到 HF，需要读中文路线时回到本页。

四个入口

每个平台只做自己最擅长的事。

这样成本最低，也最不容易把论文、开源、官网、中文传播和私有系统混成一团。

开放边界

最大诚意开放验证，严格保护核心闭环。

可以公开：概念、论文、schema、玩具 demo、非敏感样例、反例提交模板、证据阶梯、公开 DOI。 不公开：真实客户记忆、金融执行阈值、私有工具路由、生产 prompt、未公开实验、现场运行日志和商业线索。

  • 公开层让别人能验证方向，而不是复制系统。
  • 反例通道让攻击变成资产，而不是口水战。
  • GitHub 负责社区协作，HF 负责技术镜像，中文区负责解释。
  • 匿名投稿材料永远不和实名公开页混用。
