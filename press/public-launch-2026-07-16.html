A familiar voice is not a verified identity.

Artifact: voice_identity_action_gate_v1. This note separates a voice from an identity claim: if the request arrives through a familiar channel, the record still needs an independent verification phrase and an action lock before money, credentials, or control move.

Evidence Map Registries Boundaries Counterexamples

Artifact

Voice is a weak identity signal.

A deepfake or cloned voice can imitate familiarity, urgency, and tone while still carrying the wrong intent. The review object asks whether an independent channel, not the voice itself, controls the action path.

Gate

The request needs a second route before it can move money or access.

The practical boundary is simple: if the caller wants a transfer, a password reset, a credential change, or another sensitive move, the system should require a verification phrase and an action lock that does not depend on the same voice channel.

Review

If urgency shortens verification, the gate has failed.

A useful challenge should identify a missing field, an unsupported causal link, a stale evidence timestamp, or a condition under which the proposed control fails. If urgency can bypass the second channel, the action lock is not strong enough.
