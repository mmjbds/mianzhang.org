Public Release - 2026-06-05

The paper matrix should be machine-auditable.

Beyond a page that humans can browse, the 16-paper upload matrix needs a public index shape: paper_id, public file, claim, boundary, hash, DOI/status, freshness, and challenge route.

paper_id public_file claim boundary attack_route

Upload Matrix

Readers need pages. Reviewers and tools need records.

The current public record is tied to a 16-paper upload matrix: W0, P23, P28, P29, P30, P31, P32-P40, and F1/P8. The point is not paper-count promotion. The point is to make each paper's claim, evidence, boundary, freshness status, and challenge route inspectable.

The public index is a v0 export shape, not a request to expose private storage. It names the minimum fields that should be machine-readable before the upload matrix becomes a public story.

1

papers.jsonl

One row per paper or paper group, with public URL, status, hash, DOI/status, and latest verification date.

2

claims.jsonl

Claim identifiers, claim text, scope, evidence tier, does_not_claim, and boundary note.

3

evidence.jsonl

Public artifact links, manifest status, checksum status, reproduction route, and known limitation.

4

counterexamples.jsonl

Challenge routes by paper_id, target_claim, evidence_gap, class, status, and boundary effect.

5

boundaries.jsonl

Old boundary, new boundary, downgrade trigger, withdraw condition, and public note.

6

freshness

Last verified date, DOI/hash status, route health, and whether the public artifact is stale.

Public / Private Filter

Export public fields, not local context.

A useful JSONL index should be easy to crawl, diff, and review. It should not leak local absolute paths, private routes, customer information, account state, internal filenames, unpublished download links, private prompts, or execution logs.

If a field cannot be explained as public evidence, public boundary, public freshness, or public attack route, it should not enter the export.

Challenge

Audit the index before trusting the narrative.

The best critique for this artifact is a field-level critique: which row exposes non-public context, which public route cannot be traced to an upload package, which hash or DOI status is stale, or which claim lacks a usable challenge route.

The standard remains: public credibility, not public control authority.
