Translation

The external phrase is decision system, not magic intelligence.

SOVEREIGN should be explained as a system architecture for evidence, memory, tasks, boundaries, and review loops around AI-assisted decisions.

The local-first boundary matters because private user data, credentials, and execution traces should not become public evidence just because the public site discusses the architecture.

Public Layer

The public layer exposes evidence routes, not the private operating core.

The website can publish papers, concepts, demo boundaries, registries, and challenge routes.

It should not publish private orchestration, credentials, customer data, or non-public execution traces as SEO material.

FAQ

Is SOVEREIGN a chatbot?

No. It is better described as a local-first decision-system architecture around evidence, tasks, memory, and boundaries.

FAQ

Why local-first?

Because sensitive user state and execution traces should stay outside public evidence unless explicitly made public.

FAQ

What can be publicly checked?

Papers, concept entries, evidence maps, registries, issue routes, and bounded demos.
